// How It Works

Under
the hood

A precise, multi-stage pipeline from process discovery to detection result.

01
Process Discovery

Locates all active Minecraft JVM processes via system enumeration. No manual PID configuration required.

02
Memory Access

Opens the process with read permissions via the Windows API, enumerating all committed, accessible memory regions.

03
Signature Matching

Each 1 MB chunk is scanned against the full signature database. Matches are flagged instantly across MEM, JVM, DLL, JAR, and NET vectors.

04
Result Dispatch

Clean output shows every detected client, its threat category, and a confidence score — sent to Discord and ready to share.
